13. Хилерские ротации и выбор цели Хилер
Обязательно:
Для того чтобы ротация работала как хилерская, в окне ротации нужно включить соответствующую галочку хилера. После этого
программа начинает использовать специальную логику для лечения группы.
Для хилерских ротаций используются три базовых списка действий, через которые и строится вся основная логика:
СПИСОК КОГДА ИСПОЛЬЗУЕТСЯ
actions.auto_heal Автоматическое лечение в режиме AUTO
actions.manual_heal Ручное лечение дружественной текущей цели в режиме MANUAL
actions.damage Дамаг-ротация, когда лечить некого или текущая цель враждебна
Именно эти три списка являются базовыми для хилерской ротации. Вы можете создавать и любые другие дополнительные
списки, но все они должны вызываться из одного из этих трёх базовых списков.
Список default в хилерской логике использовать не следует - хилерская ротация должна строиться через auto_heal,
manual_heal и damage.
В режиме AUTO (автоматический выбор целей) программа работает по списку auto_heal. Если ничего из него не
отработало и лечить никого не нужно, то она переходит к списку damage. В режиме MANUAL (ручной выбор целей)
программа работает по списку manual_heal если выбрана дружественная цель или по списку damage если в таргете враг.
target_if — выбор цели для лечения
Параметр target_if используется для выбора нужного члена группы / рейда по указанному условию. Этот параметр
работает только в сочетании с командой call_action_list. Фактически при помощи этого параметра выбирается цель,

call_action_list — вызов списка действий
name=список — имя списка, который нужно выполнить
target_if=значение — правило выбора цели для этого списка
if=условие — дополнительное условие вызова списка
ЗНАЧЕНИЕ ЧТО ДЕЛАЕТ ПРИМЕР
min:выражениеВыбирает члена группы с минимальным значением выраженияtarget_if=min:health.pct
max:выражениеВыбирает члена группы с максимальным значением выраженияtarget_if=max:buff.renew.remains
tank Выбирает танка target_if=tank
me Выбирает вас target_if=me
need_90 Выбирает первого подходящего игрока с дебаффом из списка Heal
Above 90% Debuffs
target_if=need_90
absorb_shieldВыбирает первого подходящего игрока с дебаффом из списка Absorb
Shield Debuffs
target_if=absorb_shield
Варианты need_90 и absorb_shield работают только для хилерских профилей и только если у вас заполнены
соответствующие списки в настройках диспелов.
После target_if можно дополнительно использовать и обычное условие if=. В таком случае программа сначала
выбирает цель по правилу target_if, а затем уже проверяет условие на этой выбранной цели.
Поэтому после target_if можно использовать, например, if=target.health.pct<80 — это будет относиться не к
текущей игровой цели, а к той цели, которую только что выбрал target_if.
# Выбрать самого просевшего члена, если % его жизней менее 70%, то работать с ним по списку single_heal
actions.auto_heal+=call_action_list,name=single_heal,target_if=min:health.pct,if=target.health.pct<70
# Выбираем танка и выполняем список только если здоровье танка ниже 60%
actions.auto_heal+=call_action_list,name=tank_heal,target_if=tank,if=target.health.pct<60
# Если есть игрок с дебаффом из списка Heal Above 90% Debuffs, выбираем его и проверяем условие уже на нём
actions.auto_heal+=call_action_list,name=need_90_heal,target_if=need_90,if=target.health.pct<90
Важно:
Если список вызывается через target_if, то выбранная этим параметром цель становится текущей целью для всего
вызываемого списка. Это значит, что внутри списка все проверки вида target.*, health.pct, debuff.*, dot.* и другие
выражения, относящиеся к цели, будут работать именно с той целью, которую выбрал target_if. Проверки вида buff.* без
явного префикса по-прежнему относятся к игроку.
