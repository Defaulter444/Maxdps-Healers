8. Баффы, дебаффы и DoT
Для проверки баффов, дебаффов и DoT используется следующий формат записи:
buff / debuff / dot.название_или_ID_ауры.параметр
buff / debuff / dot — тип ауры, которую вы хотите проверить
название_или_ID_ауры — имя баффа, дебаффа или DoT в формате программы либо числовой ID способности
параметр — что именно нужно проверить: например up, remains или stacks
buff без дополнительного префикса относится к игроку. debuff и dot без дополнительного префикса относятся к
текущей цели.
По умолчанию debuff и dot на цели проверяют только ваши эффекты. Но если после названия или ID ауры указать
.any, будут учитываться эффекты от любых источников.
ПАРАМЕТР ЧТО ВОЗВРАЩАЕТ ПРИМЕР
up Аура активна if=buff.enrage.up
down Аура не активна if=debuff.772.down
ticking То же что и .up if=dot.corruption.ticking
ПАРАМЕТР ЧТО ВОЗВРАЩАЕТ ПРИМЕР
stacks Количество стаков if=buff.maelstrom_weapon.stacks>=5
remains Оставшееся время ауры if=debuff.rend.remains<4
duration Полная длительность ауры if=buff.avatar.duration>0
refreshable До окончания ауры менее 30% ее длительностиif=dot.corruption.refreshable
Проверки по ID обычно нужны в тех случаях, когда одинаковое название могут иметь разные баффы. Например, если один и
тот же бафф сначала накапливает стаки, а затем заменяется другой аурой с тем же названием, но уже с другим ID. В таких
ситуациях проверка по имени может быть недостаточно точной, поэтому лучше использовать именно ID нужной ауры.
Если нужно проверить, на скольких целях сейчас висит ваш DoT, используется отдельная запись:
active_dot.название_или_ID_dot
Например, в следующем примере Corruption будет применяться только в том случае, если этот DoT висит меньше чем на
5 целях:
actions+=corruption,if=active_dot.corruption<5
Специальные проверки дебаффов и состояний
Помимо обычных проверок по имени или ID ауры, для цели поддерживаются и специальные выражения. Они позволяют
проверять наличие снимаемых дебаффов, отдельных типов отрицательных эффектов и специальных хилерских состояний.
ВЫРАЖЕНИЕ ЧТО ВОЗВРАЩАЕТ ПРИМЕР
target.dispellable На цели есть снимаемый дебаффif=target.dispellable
target.debuff.magic.up На цели есть дебафф типа Magic if=target.debuff.magic.up
target.debuff.poison.up На цели есть дебафф типа Poison if=target.debuff.poison.up
target.debuff.disease.upНа цели есть дебафф типа Diseaseif=target.debuff.disease.up
target.debuff.curse.up На цели есть дебафф типа Curse if=target.debuff.curse.up
target.debuff.enrage.up На цели есть эффект Enrage if=target.debuff.enrage.up
target.debuff.bleed.up На цели есть эффект Bleed if=target.debuff.bleed.up
target.need_90 Цель помечена как требующая хила выше 90%if=target.need_90
target.absorb_shield На цели есть эффект поглощения входящего хилаif=target.absorb_shield
target.excluded На цели есть дебафф из списка исключений для диспелаif=!target.excluded
Параметры target.need_90, target.absorb_shield и target.excluded в первую очередь рассчитаны на хилерские
профили и связаны с настройками окна DISPEL. Они не определяются автоматически по названию эффекта, а зависят от
того, попал ли debuff в соответствующий список настроек.
То есть сами по себе эти свойства ничего не определяют автоматически — они просто проверяют, есть ли на текущей цели
дебафф, который попадает в соответствующий список из настроек.
