Типы действий
action — обычное действие
Это базовый тип действия. В строке указывается способность и, при необходимости, условия её применения.
actions+=mortal_strike
actions+=execute,if=target.health.pct<20
actions+=bloodthirst,if=rage<60
actions+=whirlwind,if=active_enemies>=3
action_list — списки действий
Когда ротация становится большой и содержит много условий, её удобно разбить на отдельные блоки (списки). Каждый
такой список содержит собственный набор действий, а в основной ротации можно вызывать уже готовый список целиком.
Например, отдельно можно сделать список для одиночной цели, для AOE, кулдаунов или защитных способностей.
Для работы со списками используется следующий формат записи:
actions.имя_списка+=название_способности,if=условие
actions — обязательное ключевое слово
имя_списка — название списка действий
название_способности — имя способности
if=условие — условие применения (необязательно)
Т.е. для добавления способностей в список используется тот же принцип записи, что и в основной ротации, но добавляется
название списка, в который мы добавляем способность. Для первой строки списка можно использовать знак =, чтобы
визуально обозначить начало нового списка. Однако это не обязательно — при желании можно использовать += для всех
строк подряд.
Формат записи выглядит так:
actions.список=первая_способность,if=условие
actions.список+=следующая_способность,if=условие
# либо можно так
actions.список+=первая_способность,if=условие
actions.список+=следующая_способность,if=условие
Например, разобьём нашу ротацию на два списка:
# Список способностей для AOE
actions.aoe=whirlwind
actions.aoe+=cleave
# Список способностей для одной цели
actions.st=mortal_strike
actions.st+=slam
В этом примере ротация разбита на два списка: один для AOE-способностей, а другой для боя с одной целью.
call_action_list — вызов списка действий
После создания списков они сами по себе не выполняются. Чтобы программа начала использовать действия из нужного
списка, его необходимо явно вызвать.
Для этого используется команда call_action_list, в которой указывается имя списка и, при необходимости, условие
вызова.
Формат записи выглядит так:
actions+=call_action_list,name=имя_списка,if=условие
actions — основной список действий
call_action_list — команда вызова списка
name=имя_списка — какой список нужно вызвать
if=условие — условие вызова списка (необязательно)
То есть вы просто добавляете вызов списка так же, как делаете это при добавлении обычных способностей. Только вместо
названия способности указывается функция call_action_list, а после name= пишется название нужного списка.
# Основной список действий
actions+=battle_shout,if=buff.battle_shout.down
actions+=call_action_list,name=aoe,if=active_enemies>=3
actions+=call_action_list,name=st,if=active_enemies<3
В этом примере сначала проверяется обычная способность Battle Shout. Если баф отсутствует, способность будет
использована. Затем, в зависимости от количества целей, вызывается список aoe или st.
Важно понимать: все строки, добавленные через actions+=... без указания имени списка, автоматически попадают в
основной список действий по умолчанию — условный default action list.
После нажатия кнопки Start программа начинает выполнять ротацию именно с этого списка, последовательно проверяя
строки, которые в нём находятся.
Если у вас есть дополнительные именованные списки, сами по себе они выполняться не будут. Их необходимо вызывать из
основного списка (либо из других уже вызванных списков) через call_action_list.
Если вы просто создадите список, но нигде его не вызовете, он останется в коде, но использоваться программой не будет.
Таким образом, основной список — это корень ротации, а остальные списки служат вспомогательными блоками логики,
которые подключаются по мере необходимости.
variable — переменные
Иногда в ротации встречаются длинные или часто повторяющиеся выражения. Чтобы не писать их заново в каждой строке,
их удобно вынести в переменную.
После этого вместо повторения одного и того же выражения можно использовать короткую запись через variable.имя.
Для работы с переменными используются два варианта записи:
actions+=variable,name=имя,value=значение
actions+=variable,name=имя,op=setif,value=значение,value_else=значение,condition=условие
actions — обязательное ключевое слово
variable — создание переменной
name=имя — название переменной
value=значение — значение, которое будет записано в переменную
op=setif — условное присваивание
value_else=значение — значение, которое будет записано, если условие не выполнено
condition=условие — условие, по которому выбирается между value и value_else
То есть переменная сама по себе ничего не применяет и не кастует — она просто сохраняет значение, которое потом можно
использовать в других строках.
Например, можно один раз сохранить часто используемую проверку в переменную, а потом использовать её в нескольких
местах:
# Сохраняем проверку в переменную
actions+=variable,name=need_cds,value=cooldown.avatar.remains<5
# Используем переменную дальше
actions+=battle_cry,if=variable.need_cds
actions+=avatar,if=variable.need_cds
В этом примере в переменную need_cds записывается результат проверки: осталось ли до готовности Avatar меньше 5
секунд. Дальше эта переменная используется в других строках, чтобы не писать одно и то же выражение повторно.
Также можно задавать переменной разные значения в зависимости от условия:
# Если Recklessness активен — используем один порог ярости, иначе другой
actions+=variable,name=slam_rage_limit,op=setif,value=80,value_else=40,condition=buff.recklessness.up
# Используем переменную в условии
actions+=slam,if=rage>variable.slam_rage_limit
Здесь переменная slam_rage_limit получает значение 80, если на вас есть баф Recklessness, и 40 — если бафа нет.
После этого это значение используется в условии для Slam.
