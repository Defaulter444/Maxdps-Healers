18. Диспелы Хилер
Окно ⚕  DISPEL доступно у хилерских профилей и работает не только как настройка авто-диспела, но и как источник
нескольких специальных флагов для хилерской логики. Все поля принимают Spell ID дебаффов. ID можно указывать по
одному в строке, через пробел или через запятую.
дебафф на цели→сверяется со списками настроек→влияет на диспел и выбор цели
Exception Debuffs — не снимать эти дебаффы
Heal Above 90% Debuffs — помечать цель как требующую лечения выше 90%
Absorb Shield Debuffs / Exclude Debuffs — влиять на приоритет и исключение цели из лечения
ПОЛЕ ЧТО ДЕЛАЕТ В КОДЕ СВЯЗАННЫЕ ВЫРАЖЕНИЯ
Exception
Debuffs
Дебаффы из этого списка считаются исключениями для
диспела. Они не должны сниматься вручную через ваш
диспел-спелл.
target.excluded здесь не используется; это
именно список исключений для диспела.
Heal Above
90% Debuffs
Если на члене группы есть такой дебафф, цель получает флаг
need_90. Это повышает её приоритет в хиле даже если
визуально HP уже высокий.
target.need_90, target_if=need_90,
raid.has_need_90, raid.count_need_90
Absorb
Shield
Debuffs
Если на цели есть такой дебафф, она получает флаг
absorb_shield. Для расчёта эффективного HP цель
дополнительно занижается до 70%, чтобы лечение выбирало
её агрессивнее.
target.absorb_shield,
target_if=absorb_shield,
raid.has_absorb_shield,
raid.count_absorb_shield
Exclude
Debuffs
Если на цели есть такой дебафф, она считается исключённой
из пула лечения. Такие цели пропускаются в ряде рейдовых
проверок и при fixed-target выборе need_90 /
absorb_shield.
target.excluded
Как это работает при касте диспела
Приложение дополнительно проверяет сам диспел-спелл в рантайме. Если способность действительно является диспелом
по DB2 и у выбранного юнита нет подходящего снимаемого дебаффа нужного типа, такая строка ротации будет
автоматически пропущена.
Важно:
проверка идёт по типу диспела, который умеет снимать сам спелл. Сейчас автоматически учитываются типы Curse, Disease,
Magic и Poison.
В healer-контексте проверяется текущая цель хила, выбранная через target_if или другой механизм выбора союзника. В
обычном DPS / Manual-контексте такая проверка выполняется на самом игроке, то есть для само-диспела.
Выражения, связанные с диспелом
ВЫРАЖЕНИЕ ЧТО ВОЗВРАЩАЕТ ПРИМЕР
target.dispellable На цели есть хотя бы один снимаемый дебаффif=target.dispellable
target.debuff.magic.upНа цели есть harmful-аура типа Magicif=target.debuff.magic.up
target.debuff.poison.upНа цели есть harmful-аура типа Poison if=target.debuff.poison.up
target.debuff.disease.upНа цели есть harmful-аура типа Diseaseif=target.debuff.disease.up
target.debuff.curse.upНа цели есть harmful-аура типа Curseif=target.debuff.curse.up
target.need_90 На цели есть дебафф из списка Heal Above 90% Debuffsif=target.need_90
target.absorb_shield На цели есть дебафф из списка Absorb Shield Debuffsif=target.absorb_shield
target.excluded На цели есть дебафф из списка Exclude Debuffs if=!target.excluded
raid.has_dispellable В группе есть хотя бы один живой и не исключённый союзник со
снимаемым дебаффом
if=raid.has_dispellable
raid.count_dispellableКоличество союзников со снимаемыми дебаффамиif=raid.count_dispellable>0
Практический смысл:
если вы добавили в ротацию диспел-спелл, обычно не нужно вручную городить сложное условие на каждый тип дебаффа.
Достаточно выбрать правильную цель и при необходимости ограничить её через target.excluded, target.need_90 или
рейдовые проверки.
